C++ Programming Language

Introduction

C++ is a general-purpose, object-oriented programming language developed by Bjarne Stroustrup in 1979. It extends the C language with features like classes, inheritance, polymorphism, and templates.

What is C++?

C++ is widely used for system programming, operating systems, game engines, and applications requiring high performance. It balances low-level memory control with high-level abstractions.

Why Learn C++?

1. Foundation of Modern Programming

Definition: C++ is one of the oldest yet most powerful programming languages, forming the basis of many modern languages and systems.

Explanation: Learning C++ gives you a strong grasp of programming fundamentals like memory management, object-oriented design, and efficient algorithms. Once you understand C++, other languages (like Java, Python, or Rust) feel easier.

2. Performance & Efficiency

Definition: C++ is a compiled language that provides direct control over hardware and memory.

Explanation: Unlike interpreted languages, C++ programs run extremely fast and are used in performance-critical applications such as operating systems, game engines, and financial systems.

3. Object-Oriented Programming (OOP)

Definition: OOP is a paradigm that organizes code into classes and objects.

Explanation: C++ introduced OOP features like encapsulation, inheritance, and polymorphism. These concepts are essential for building scalable, maintainable software.

4. System-Level Programming

Definition: System programming involves writing software that interacts closely with hardware and operating systems.

Explanation: C++ allows low-level memory manipulation using pointers, making it ideal for embedded systems, device drivers, and OS kernels.

5. Standard Template Library (STL)

Definition: STL is a collection of pre-written classes and functions for data structures and algorithms.

Explanation: With STL, you can quickly implement vectors, maps, sets, and algorithms like sorting and searching without writing them from scratch.

6. Industry Demand

Definition: C++ remains one of the most in-demand languages in industries requiring high performance.

Explanation: Fields like game development (Unreal Engine), finance (quantitative trading systems), and robotics heavily rely on C++.

7. Gateway to Competitive Programming

Definition: Competitive programming is solving algorithmic problems under time constraints.

Explanation: C++ is the most popular language in competitive programming because of its speed, STL support, and flexibility.

8. Longevity & Community

Definition: C++ has been evolving for decades with modern standards (C++11, C++14, C++17, C++20).

Explanation: A massive global community ensures continuous updates, libraries, and support, making it a safe long-term investment for learners.

Conclusion: Learning C++ gives you deep control, strong fundamentals, and access to high-performance applications. It’s like learning the “engine room” of programming before moving to higher-level languages.

Abstraction in C++

What is Abstraction in C++?

Definition: Abstraction is the process of exposing only necessary details to the user and hiding the background implementation.

Purpose: Simplifies code, improves maintainability, and enhances modularity.

Real-life Analogy:Driving a car—you know how to use the steering wheel, accelerator, and brakes, but you don’t need to know the internal mechanics of how they work.

Types of Abstraction in C++

1. Data Abstraction

  • Achieved using classes with private data members and public methods.

Example:


class Student {
private:
    int marks;   // hidden data
public:
    void setMarks(int m) { marks = m; }
    int getMarks() { return marks; }
};

Here, marks is hidden, and only controlled access is provided through setMarks() and getMarks().

2. Control Abstraction

  • Achieved using functions and abstract classes.
  • Focuses on hiding the implementation of operations while exposing only the interface.

How Abstraction is Implemented?

  • Access Specifiers: private hides details, public exposes necessary methods.
  • Abstract Classes: A class with at least one pure virtual function.
    • 
class Shape {
public:
    virtual double area() = 0;  // pure virtual function
};

class Rectangle : public Shape {
    double length, width;
public:
    Rectangle(double l, double w) : length(l), width(w) {}
    double area() override { return length * width; }
};

    Shape defines what must be done (area()), while Rectangle defines how it is done.

  • Interfaces (Full Abstraction):
    • A class with only pure virtual functions and no data members.
    • Forces all derived classes to implement every function.

Benefits of Abstraction

  • Encapsulation of complexity: Users interact with simple interfaces.
  • Improved maintainability: Internal changes don’t affect external code.
  • Flexibility & scalability: Easier to extend systems with new classes.
  • Security: Sensitive data can be hidden from direct access.

Encapsulation in C++

What is Encapsulation in C++?

Definition: Encapsulation is one of the core principles of Object-Oriented Programming (OOP) in C++. It refers to wrapping data (variables) and methods (functions) together into a single unit called a class, while restricting direct access to the internal details.

Encapsulation in C++ means binding data and the code that manipulates it together and controlling access using access specifiers ( private, protected, public ).

  • Private/Protected members → hidden from outside interference.
  • Public methods → provide controlled access to the hidden data.

Purpose

  • Protects data from accidental modification.
  • Provides a clean interface for interaction.
  • Improves modularity and maintainability.
  • Ensures security by hiding sensitive details.

Example of Encapsulation

#include <iostream> 
using namespace std;

class BankAccount {
private:
    double balance;  // hidden data

public:
    BankAccount(double initialBalance) {
        balance = initialBalance;
    }

    void deposit(double amount) {
        if(amount > 0) balance += amount;
    }

    void withdraw(double amount) {
        if(amount <= balance) balance -= amount;
        else cout << "Insufficient funds!" << endl;
    }

    double getBalance() {
        return balance;
    }
};

int main() {
    BankAccount account(1000);   // create account with initial balance
    account.deposit(500);        // controlled access
    account.withdraw(200);
    cout << "Current Balance: " << account.getBalance() << endl;
    return 0;
}

Key Points in Example

  • balance is private → cannot be accessed directly outside the class.
  • deposit(), withdraw(), and getBalance() are public methods → provide controlled
  • This ensures data safety and controlled modification.

Benefits of Encapsulation

  • Data Hiding: Prevents unauthorized access.
  • Flexibility: Internal implementation can change without affecting external code.
  • Reusability: Classes can be reused in different programs.
  • Security: Sensitive data is protected.

✅ In short: Encapsulation in C++ is about hiding the internal details of a class and exposing only necessary functionality through public methods. It makes programs more secure, modular, and easier to maintain.

Inheritance in C++

What is Inheritance in C++

Definition: Inheritance is a fundamental concept of Object-Oriented Programming (OOP) that allows one class (called the derived class) to acquire the properties and behaviors of another class (called the base class).

It promotes code reusability and hierarchical classification.

Inheritance in C++ is the process by which a new class is created from an existing class, enabling the derived class to reuse and extend the functionality of the base class.


class DerivedClass : access_specifier BaseClass {
    // additional members
};

Basic Example

#include <iostream>
using namespace std;

class Animal {
public:
    void eat() {
        cout << "Eating..." << endl;
    }
};

class Dog : public Animal {
public:
    void bark() {
        cout << "Barking..." << endl;
    }
};

int main() {
    Dog d;
    d.eat();   // inherited from Animal
    d.bark();  // defined in Dog
    return 0;
}

Types of Inheritance in C++

Type Description Example
Single Inheritance One derived class inherits from one base class. class B : public A { };
Multiple Inheritance One derived class inherits from more than one base class. class C : public A, public B { };
Multilevel Inheritance A class is derived from another derived class. class C : public B; class B : public A;
Hierarchical Inheritance Multiple classes inherit from a single base class. class B : public A; class C : public A;
Hybrid Inheritance Combination of two or more types of inheritance. Mix of multiple + multilevel.

1. Single Inheritance

A derived class inherits from only one base class.

#include <iostream>
using namespace std;

class Parent {
public:
    void display() {
        cout << "Parent class function" << endl;
    }
};

class Child : public Parent {
public:
    void show() {
        cout << "Child class function" << endl;
    }
};

int main() {
    Child obj;
    obj.display();
    obj.show();
    return 0;
}

2. Multiple Inheritance

A derived class inherits from more than one base class.

#include <iostream>
using namespace std;

class A {
public:
    void msgA() { cout << "Class A" << endl; }
};

class B {
public:
    void msgB() { cout << "Class B" << endl; }
};

class C : public A, public B {
public:
    void msgC() { cout << "Class C" << endl; }
};

int main() {
    C obj;
    obj.msgA();
    obj.msgB();
    obj.msgC();
    return 0;
}

3. Multilevel Inheritance

A class is derived from another derived class (a chain of inheritance).

#include <iostream>
using namespace std;

class GrandParent {
public:
    void funcGP() { cout << "GrandParent class" << endl; }
};

class Parent : public GrandParent {
public:
    void funcP() { cout << "Parent class" << endl; }
};

class Child : public Parent {
public:
    void funcC() { cout << "Child class" << endl; }
};

int main() {
    Child obj;
    obj.funcGP();
    obj.funcP();
    obj.funcC();
    return 0;
}

4. Hierarchical Inheritance

Multiple derived classes inherit from a single base class.

#include <iostream>
using namespace std;

class Base {
public:
    void showBase() { cout << "Base class" << endl; }
};

class Derived1 : public Base {
public:
    void showD1() { cout << "Derived1 class" << endl; }
};

class Derived2 : public Base {
public:
    void showD2() { cout << "Derived2 class" << endl; }
};

int main() {
    Derived1 d1;
    Derived2 d2;
    d1.showBase();
    d2.showBase();
    return 0;
}

5. Hybrid Inheritance

A combination of two or more types of inheritance (e.g., multiple + hierarchical). This often leads to the diamond problem.

#include <iostream>
using namespace std;

class A {
public:
    void msgA() { cout << "Class A" << endl; }
};

class B : public A {
public:
    void msgB() { cout << "Class B" << endl; }
};

class C {
public:
    void msgC() { cout << "Class C" << endl; }
};

class D : public B, public C {
public:
    void msgD() { cout << "Class D" << endl; }
};

int main() {
    D obj;
    obj.msgA();
    obj.msgB();
    obj.msgC();
    obj.msgD();
    return 0;
}

Key Advantages

  • Code Reusability
  • Reduced Redundancy
  • Improved Maintainability
  • Logical Hierarchy Representation

Polymorphism in C++

What is Polymorphism in C++

Polymorphism is one of the four pillars of Object-Oriented Programming (OOP) in C++. It means “many forms” — allowing the same function or operator to behave differently based on the context.

Polymorphism in C++ is the ability of a function, method, or operator to take multiple forms depending on the type of object or arguments involved. It enables code flexibility, reusability, and dynamic behavior.

Types of Polymorphism in C++

Type Description Example
Compile-time Polymorphism Achieved during compilation using function overloading or operator overloading. Function Overloading, Operator Overloading
Run-time Polymorphism Achieved during program execution using virtual functions and inheritance. Virtual Functions

1. Compile-time Polymorphism

(a) Function Overloading

Multiple functions with the same name but different parameter lists.


#include <iostream>
using namespace std;

class Print {
public:
    void show(int i) {
        cout << "Integer: " << i << endl;
    }
    void show(double d) {
        cout << "Double: " << d << endl;
    }
    void show(string s) {
        cout << "String: " << s << endl;
    }
};

int main() {
    Print p;
    p.show(10);
    p.show(3.14);
    p.show("Hello C++");
    return 0;
}

(b) Operator Overloading

Redefining the behavior of operators for user-defined types.


#include <iostream>
using namespace std;

class Complex {
    int real, imag;
public:
    Complex(int r=0, int i=0) { real = r; imag = i; }

    Complex operator + (Complex const &obj) {
        return Complex(real + obj.real, imag + obj.imag);
    }

    void display() {
        cout << real << " + " << imag << "i" << endl;
    }
};

int main() {
    Complex c1(3, 4), c2(1, 2);
    Complex c3 = c1 + c2;
    c3.display();
    return 0;
}

2. Run-time Polymorphism

Virtual Functions

Allows derived classes to override base class methods dynamically.


#include <iostream>
using namespace std;

class Base {
public:
    virtual void show() {
        cout << "Base class show()" << endl;
    }
};

class Derived : public Base {
public:
    void show() override {
        cout << "Derived class show()" << endl;
    }
};

int main() {
    Base* bptr;
    Derived d;
    bptr = &d;
    bptr->show();  // Calls Derived's version
    return 0;
}

Key Differences

Feature Compile-time Polymorphism Run-time Polymorphism
Binding Time At compile time At runtime
Techniques Function & Operator Overloading Virtual Functions
Speed Faster Slightly slower
Flexibility Limited High
Inheritance Required No Yes

Advantages

  • Code Reusability
  • Flexibility and Scalability
  • Easier Maintenance
  • Supports Dynamic Behavior

Difference between C and C++

Review of C

C Language is a procedural programming language developed in the 1970s. It focuses on functions and procedures rather than objects.

  • Structured programming (functions, loops, conditionals)
  • Manual memory management using malloc() and free()
  • Input/Output handled by printf() and scanf()
  • No built-in support for OOP concepts like classes, inheritance, or polymorphism

Difference between C and C++

Feature C C++
Paradigm Procedural Procedural + Object-Oriented
Input/Output printf, scanf cin, cout
Memory Management malloc, free new, delete
Data & Functions Separate Encapsulated in classes
Operators Fixed meaning Supports operator overloading
OOP Concepts Not supported Supported (Abstraction, Encapsulation, Inheritance, Polymorphism)

    int x;

Classes and Objects in C++

Classes and objects are the foundation of Object-Oriented Programming (OOP) in C++. A class is a blueprint that defines attributes and behaviors, while an object is an instance of that class.

Classes

A class contains data members (variables) and member functions (methods). Think of it as a design plan.

Object

An object is created from a class and represents a real-world entity. If the class is the blueprint, the object is the actual product.

Example: Class & Object


#include <iostream>
using namespace std;

class Car {
public:
    string brand;
    int year;

    void display() {
        cout << "Brand: " << brand << ", Year: " << year << endl;
    }
};

int main() {
    Car car1;           // object creation
    car1.brand = "Tesla";
    car1.year = 2024;

    car1.display();     // accessing class method
    return 0;
}

Information Hiding

Information hiding restricts direct access to internal details of a class. It is achieved using private, protected, and public specifiers. This ensures data security and prevents misuse.


class Student {
private:
    int marks;   // hidden data

public:
    void setMarks(int m) {
        if(m >= 0) marks = m;
    }
    int getMarks() {
        return marks;
    }
};

Abstract Data Types

An Abstract Data Type (ADT) is defined by its behavior (operations) rather than its implementation. In C++, classes are used to implement ADTs such as Stack, Queue, and Linked List.


class Stack {
private:
    int arr[5];
    int top;

public:
    Stack() { top = -1; }

    void push(int x) {
        if(top < 4) arr[++top] = x;
    }

    int pop() {
        if(top >= 0) return arr[top--];
        return -1; // error value
    }
};

Attributes

Attributes are variables inside a class that represent the state or properties of an object. Example: brand and year in the Car class.

Methods

Methods are functions inside a class that define the behavior of an object.


class BankAccount {
private:
    double balance;

public:
    BankAccount(double initial) { balance = initial; }
    void deposit(double amount) { balance += amount; }
    double getBalance() { return balance; }
};

Class Declaration

A class declaration defines the structure of a class in C++. It specifies attributes (data members), methods (member functions), and access specifiers. It acts as a blueprint for creating objects and enforces encapsulation.


class Car {
private:
    string brand;
    int year;

public:
    void setData(string b, int y) {
        brand = b; year = y;
    }
    void display() {
        cout << brand << " (" << year << ")" << endl;
    }
};

Object Identity, State & Behavior

  • Identity → Unique reference (object name).
  • State → Values stored in attributes.
  • Behavior → Actions performed via methods.

Together, these make an object a self-contained unit in OOP.


class Car {
public:
    string brand;   // state
    int year;       // state

    void display() {   // behavior
        cout << brand << " (" << year << ")" << endl;
    }
};

int main() {
    Car c1;   // identity
    c1.brand = "Tesla";
    c1.year = 2024;
    c1.display();
}

Constructors

A constructor is a special member function used to initialize objects. It has the same name as the class, no return type, and is called automatically when an object is created.

Types of constructors:

  • Default Constructor → No parameters, initializes with default values.
  • Parameterized Constructor → Accepts arguments to initialize attributes.
  • Copy Constructor → Creates a new object as a copy of another.
Default Constructor

Example

    }
};
Parameterized Constructor

Example


class Student {

Copy Constructor

Example

Destructor

A destructor is a special member function used to clean up resources when an object is destroyed. It has the same name as the class, preceded by ~. It is called automatically when an object goes out of scope.

Example


class Demo {
public:
    Demo() { cout << "Constructor called!" << endl; }
    ~Demo() { cout << "Destructor called!" << endl; }
};

int main() {
    Demo d1;   // constructor runs
}   // destructor runs automatically

Instantiation of Objects

Instantiation refers to the creation of a runtime instance of a class. In C++, this can occur in two primary memory areas:

  • Stack Allocation: Memory is managed automatically. The object is destroyed when it goes out of scope.
  • Heap Allocation: Controlled via the new keyword. The developer must manage the lifecycle.

MyClass obj;            // Stack: Automatic
MyClass* ptr = new MyClass(); // Heap: Manual

Default Parameter Values

C++ allows functions to have default values for arguments. If a value isn't provided by the caller, the default is used.

Rule: Default arguments must be assigned from right to left in the function signature. 

void display(int width, int height = 100); // Valid
// void display(int width = 50, int height); // Invalid

Object Types

Objects are categorized based on their complexity and memory layout:

  • POD (Plain Old Data): Simple structures compatible with C layouts.
  • Polymorphic Objects: Classes containing at least one virtual function. These utilize a vtable for dynamic dispatch.

C++ Garbage Collection

C++ does not feature a background Garbage Collector (GC) like Java. Instead, it relies on RAII (Resource Acquisition Is Initialization).

Modern C++ uses Smart Pointers (std::unique_ptr, std::shared_ptr) to automate memory release through reference counting and ownership logic, providing the benefits of GC without the performance "stop-the-world" overhead.

Abstract Classes & Metaclasses

Abstract Classes

A class designed to be an interface. It cannot be instantiated and must contain at least one pure virtual function.

virtual void signal() = 0;

Metaclasses

While C++ doesn't have native metaclasses like Python, it uses Template Metaprogramming (TMP) to generate class definitions at compile-time, serving a similar structural purpose.

Inheritance and Class Hierarchy

Definition: Inheritance allows a class (Derived Class) to acquire the properties and methods of another class (Base Class). A Class Hierarchy is the resulting tree-like structure of these relationships.

Derivation Modes

  • Public: Public members of the base class remain public in the derived class.
  • Protected: Public members become protected; protected members remain protected.
  • Private: All members of the base class become private in the derived class. 

class Employee {
protected:
    string name;
public:
    Employee(string n) : name(n) {}
};

class Developer : public Employee {
public:
    Developer(string n) : Employee(n) {}
    void code() { cout << name << " is coding..."; }
};

Aggregation, Composition, and Classification

These terms define how objects relate to one another beyond simple inheritance.

  • Classification Hierarchy: The standard "is-a" relationship (e.g., A Square is a Shape).
  • Aggregation: A "has-a" relationship where the child can exist independently (e.g., A University has Professors).
  • Composition: A "has-a" relationship where the child's lifecycle depends on the parent (e.g., A Building has Floors).

Polymorphism Techniques

Definition: The ability of a single interface to represent different underlying forms (data types or functions).

Method Polymorphism

Consists of Overloading (same name, different parameters) and Overriding (redefining a base method in a child class).

Parametric Polymorphism

Also known as Generics or Templates. It allows a function or class to work with any data type without rewriting code.

// Template Example
template <typename T>
T multiply(T a, T b) {
    return a * b;
}

Generic Functions

Definition: A function that is designed to work with any data type. Instead of writing separate functions for int, float, and double, you write one "generic" version.

  • Explanation: This is a form of Parametric Polymorphism. Instead of defining multiple functions for int, float, etc., you define one template. The compiler generates the specific code only when the function is called with a specific type.
  • Code Example:
#include <iostream>
using namespace std;

// Generic function using templates
template <typename T>
T findMax(T a, T b) {
    return (a > b) ? a : b;
}

int main() {
    cout << findMax(5, 10) << endl;       // Works with int
    cout << findMax(5.5, 2.1) << endl; // Works with double
    return 0;
}

Function Name Overloading

Definition: Having multiple functions with the same name but different signatures (different parameters or types).

Explanation: This is Compile-time Polymorphism. It allows a single action name to be used for different types of inputs, making the code more intuitive.

void print(int i) { cout << "Integer: " << i; }
void print(string s) { cout << "String: " << s; }

Overriding Inheritance Methods

Definition: Redefining a base class method in a derived class to provide specialized behavior.

Explanation: To enable overriding, the base class must declare the function as virtual. The child class then uses the override keyword to signal it is replacing the parent's logic. 

class Base {
public:
    virtual void show() { cout << "Base Logic"; }
};

class Derived : public Base {
public:
    void show() override { cout << "Child Logic"; }
};

Run-time Polymorphism

Definition: Determining which function implementation to execute while the program is running, rather than at compile time.

Explanation: This is achieved using base class pointers and virtual functions. It allows a collection (like a list of Shape pointers) to trigger the correct draw() method for circles, squares, or triangles dynamically.

Multiple Inheritance

Definition: A feature where a class can inherit features from more than one parent class.

Explanation: This is useful for combining behaviors (e.g., a SmartPhone inherits from both Camera and Phone). However, it requires care to avoid the "Diamond Problem," where the compiler gets confused by duplicate inherited members.

class Battery { public: void power() {} };
class Processor { public: void compute() {} };

class Laptop : public Battery, public Processor { 
    // Inherits both power() and compute()
};

Streams and Files

Definition: A Stream is a sequence of bytes used as an interface to transfer data between a program and an external destination, like a File.

Key Libraries: <fstream> provides ifstream (reading), ofstream (writing), and fstream (both). 
#include <fstream>
using namespace std;

void saveRecord() {
    ofstream outFile("log.txt", ios::app); // Open in append mode

Namespaces

Definition: A mechanism to group related entities (classes, functions) under a unique name to prevent Naming Collisions.

This is essential when integrating multiple third-party libraries that might use the same function names like init() or process().

namespace Database {
    void connect() { /* SQL Logic */ }
}

namespace API {
    void connect() { /* REST Logic */ }
}

// Usage: Database::connect(); API::connect();

Exception Handling

Definition: A structured way to react to exceptional circumstances (runtime errors) during program execution.

  • Try: Block where error-prone code resides.
  • Throw: Used to signal the occurrence of an error.
  • Catch: The handler that takes action after an error.
try {
    if (divisor == 0) throw "DivisionByZeroError";
    result = total / divisor;
} catch (const char* e) {
    cout << "Critical Error: " << e;
}

Generic Classes

Definition: A class that uses Templates to allow it to operate on various data types without being rewritten for each one.

This is the foundation of data structures like Stacks, Queues, and Linked Lists in modern libraries. 

template <class T>
class Storage {
    T item;
public:
    void set(T i) { item = i; }
    T get() { return item; }
};

// Usage: Storage<int> myInts; Storage<string> myStrings;
